# 新版進銷存系統部署說明

本文說明如何將 `SmartIMS.Web` 部署到 Windows/IIS，並建立 SQL Server 資料庫。

## 1. 系統需求

- Windows Server 或 Windows 10/11
- IIS，並啟用 ASP.NET Core Hosting Bundle
- .NET 10 Runtime 或 Hosting Bundle
- SQL Server 2019 以上，或可連線的 SQL Server 執行個體
- 部署帳號需具備網站目錄讀取權限
- 首次建置資料庫的 SQL 帳號需具備 `CREATE DATABASE`、`CREATE TABLE`、`ALTER TABLE`、`CREATE INDEX` 權限

## 2. 專案結構

- 原始碼：`SmartIMS.Web/`
- 已發佈網站檔案：`Site/`
- SQL 建置腳本：`database/create_smartims_database.sql`
- 主要設定來源：環境變數

程式啟動時會執行 `DatabaseBootstrapper.EnsureAsync()`，可自動建立資料庫、資料表、索引、外鍵與預設資料。若正式環境希望先由 DBA 建庫，請先執行本文提供的 SQL 腳本。

## 3. 建立 SQL Server 資料庫

使用 SSMS 開啟 `database/create_smartims_database.sql`。

若使用 SSMS，請先啟用：

```text
Query -> SQLCMD Mode
```

如需改資料庫名稱，修改腳本第一行：

```sql
:setvar DatabaseName "SmartIMS_NewVersion"
```

執行後會建立：

- 資料庫：`SmartIMS_NewVersion`
- 使用者表：`dbo.AppUser`
- 權限表：`dbo.AppPermission`
- 使用者權限表：`dbo.UserPermission`
- 系統設定表：`dbo.SystemSetting`
- 清單偏好設定表：`dbo.UserListViewSetting`
- 稽核表：`dbo.AuditLog`
- 預設管理員：`Admin`

預設管理員密碼不直接寫在 SQL 腳本內。第一次啟動網站時，若 `Admin.PasswordHash` 為空，程式會使用 `SMARTIMS_ADMIN_PASSWORD` 環境變數設定密碼；若未設定，預設為 `admin`。

## 4. 環境變數設定

建議在 IIS Application Pool 或系統環境變數中設定：

```powershell
setx DB_HOST "localhost"
setx DB_PORT "1433"
setx DB_NAME "SmartIMS_NewVersion"
setx DB_USER "smartims_user"
setx DB_PASSWORD "your_password"
setx DB_ENCRYPT "false"
setx DB_TRUST_SERVER_CERT "true"
setx SMARTIMS_ADMIN_PASSWORD "change_this_password"
```

也可以用完整連線字串：

```powershell
setx MSSQL_CONNECTION_STRING "Server=localhost;Database=SmartIMS_NewVersion;User ID=smartims_user;Password=your_password;Encrypt=False;TrustServerCertificate=True;"
```

若 `DB_USER` 或 `DB_PASSWORD` 未設定，程式會改用 Windows Integrated Security。

## 5. 發佈網站

在專案根目錄執行：

```powershell
dotnet publish SmartIMS.Web\SmartIMS.Web.csproj -c Release -o Site
```

確認 `Site/` 內包含：

- `SmartIMS.Web.dll`
- `SmartIMS.Web.exe`
- `web.config`
- `appsettings.json`
- `wwwroot/`

## 6. IIS 部署

1. 安裝 ASP.NET Core Hosting Bundle。
2. 建立 IIS Site，例如 `SmartIMS`。
3. Physical path 指向 `E:\新版進銷存系統\Site`。
4. Application Pool 設定：
   - .NET CLR Version：`No Managed Code`
   - Pipeline mode：`Integrated`
5. 設定網站或 Application Pool 的環境變數。
6. 確認 Application Pool identity 對 `Site/` 具有讀取權限。
7. 若要上傳 Logo 或登入圖，確認 `Site/wwwroot/uploads/branding` 可寫入。
8. 重新啟動網站。

## 7. 首次登入

預設帳號：

```text
Admin
```

密碼來源：

- 優先使用 `SMARTIMS_ADMIN_PASSWORD`
- 未設定時為 `admin`

首次登入後請立即修改預設密碼或透過資料庫更新密碼流程重設。

## 8. 驗證項目

部署後請檢查：

- 網站首頁可開啟
- 可使用 `Admin` 登入
- 使用者管理頁可開啟
- 權限設定頁可開啟
- 品牌設定可儲存
- SQL Server 內有 `AppUser`、`AppPermission`、`SystemSetting` 等資料表

## 9. 常見問題

### 連線資料庫失敗

檢查 `DB_HOST`、`DB_PORT`、`DB_NAME`、`DB_USER`、`DB_PASSWORD` 或 `MSSQL_CONNECTION_STRING` 是否正確。

### SSL 或憑證錯誤

內部測試環境可設定：

```powershell
setx DB_ENCRYPT "false"
setx DB_TRUST_SERVER_CERT "true"
```

正式環境建議使用可信任憑證並啟用加密。

### 權限不足無法自動建庫

由 DBA 先執行 `database/create_smartims_database.sql`，再讓網站使用只具備日常讀寫權限的帳號連線。

### 預設密碼不是預期值

只有當 `dbo.AppUser.PasswordHash` 為空時，程式啟動才會套用 `SMARTIMS_ADMIN_PASSWORD`。若已經有密碼雜湊，請透過後台或資料庫流程重設。
